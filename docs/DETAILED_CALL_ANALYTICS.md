# Detailed Call Analytics Endpoint

## Overview

A comprehensive call analytics endpoint that provides detailed call information including caller details, call history, AI-generated insights, and full transcripts.

## Endpoint

```
GET /admin/analytics/detailed/{callId}
```

**Authentication:** Cognito User Pools (JWT token required)

**Authorization:** User must have access to the clinic associated with the call

## Response Format

```typescript
{
  // Header Information
  clinicId: string;
  clinicName: string;
  callerName: string | null;  // Extracted from conversation
  direction: "INBOUND" | "OUTBOUND";
  to: string;                  // Phone number
  from: string;                // Phone number
  callLength: number;          // Duration in seconds
  
  // Call History - Past calls from this number
  callHistory: [
    {
      callPath: string;       // "from X to Y"
      date: string;           // "MM/DD/YY"
      time: string;           // "HH:MM:SS"
      duration: string;       // "HH:MM:SS"
      typeOfCall: string;     // "inbound", "outbound", "callback", "VIP"
    }
  ],
  
  // Call Insights - AI-generated analysis
  insights: {
    summary: string;                    // AI-generated call summary
    missedOpportunity: "yes" | "no";   // Did we miss a sales opportunity?
    missedOpportunityReason: string | null;
    appointmentStatus: "scheduled" | "not_scheduled" | "rescheduled" | "cancelled" | "unknown";
    notSchedulingReason: string | null;
    billingConcerns: "yes" | "no";
    givenFeedback: "yes" | "no";
    priority: "high" | "medium" | "low";
    inquiredServices: "yes" | "no";    // Did they ask about specific services?
    callType: "appointment" | "inquiry" | "complaint" | "billing" | "emergency" | "other";
  },
  
  // Transcript - Timestamped conversation
  transcript: [
    {
      timestamp: string;      // "HH:MM:SS"
      speaker: "AGENT" | "CUSTOMER";
      text: string;
    }
  ]
}
```

## Data Sources

The endpoint aggregates data from multiple sources:

1. **CallQueue Table** - Call metadata (direction, duration, phone numbers)
2. **Clinics Table** - Clinic name
3. **Call History** - Past 30 days of calls from the same phone number
4. **RecordingMetadata Table** - Transcript data
5. **Chat History Table (chatbot-conversations)** - AI conversation analysis for insights
6. **S3 Recordings Bucket** - Full transcripts (if not in DynamoDB)

## Insights Generation

The `insights` object is automatically generated by analyzing the chat conversation:

### Appointment Status Detection
- **scheduled**: Contains keywords like "appointment scheduled", "booked successfully"
- **rescheduled**: Contains "rescheduled"
- **cancelled**: Contains "cancelled"
- **not_scheduled**: Indicated intent but no confirmation
- **unknown**: No clear appointment discussion

### Missed Opportunity Detection
Flagged as "yes" when:
- Caller expressed intent to schedule (keywords: "appointment", "schedule", "book", "emergency")
- AND appointment was not successfully scheduled
- Reason is provided explaining why (no slots, closed, no alternatives offered)

### Priority Calculation
- **high**: Emergency calls, urgent requests, unmet needs
- **medium**: Appointment requests, billing inquiries
- **low**: General inquiries, information requests

### Call Type Classification
Based on conversation keywords:
- **emergency**: "emergency", "urgent", "pain"
- **appointment**: "appointment", "schedule", "book"
- **billing**: "billing", "insurance", "payment"
- **complaint**: "complaint", "unhappy"
- **inquiry**: "question", "how much", general questions
- **other**: Default if no clear match

## Implementation

### Lambda Function
**Location:** `src/services/chime/get-detailed-call-analytics.ts`

**Environment Variables:**
- `CALL_QUEUE_TABLE_NAME` - Call records table
- `RECORDING_METADATA_TABLE_NAME` - Recording and transcript metadata
- `CHAT_HISTORY_TABLE_NAME` - Chatbot conversation history
- `CLINICS_TABLE_NAME` - Clinic information
- `RECORDINGS_BUCKET_NAME` - S3 bucket for recordings
- `COGNITO_REGION` - AWS region for Cognito
- `USER_POOL_ID` - Cognito User Pool ID

### Infrastructure

**Admin Stack** (`src/infrastructure/stacks/admin-stack.ts`)
- Lambda function created with required permissions
- API Gateway route added: `GET /analytics/detailed/{callId}`
- Grants read access to all required tables and S3 bucket

**Dependencies:**
- ChimeStack (for CallQueue, Clinics, RecordingMetadata tables)
- ChatbotStack (for conversation history)
- Analytics tables (for historical data)

```bash

  https://api.todaysdentalinsights.com/admin/analytics/detailed/call-12345 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Troubleshooting

### Common Issues

**Issue**: 404 Call Not Found
- **Cause**: CallId doesn't exist or wrong ID format
- **Fix**: Check CallQueue table for correct callId or pstnCallId

**Issue**: Empty call history
- **Cause**: Phone number index not queried correctly
- **Fix**: Verify phoneNumber-queueEntryTime-index exists on CallQueue table

**Issue**: No insights generated
- **Cause**: Chat history table not accessible or no session data
- **Fix**: Check CHAT_HISTORY_TABLE_NAME environment variable and session linkage

**Issue**: Missing transcript
- **Cause**: Recording not processed or S3 permissions issue
- **Fix**: Verify RecordingProcessor Lambda ran successfully and S3 read permissions

**Issue**: Unauthorized errors
- **Cause**: User doesn't have access to the clinic
- **Fix**: Check user's Cognito groups and clinic permissions
