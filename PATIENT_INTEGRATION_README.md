# Patient Data Integration - Quick Reference

## What Was Added

✅ **Automatic Patient Identification**: When a call completes, the system automatically identifies the patient using:
- PatNum from call metadata (if available)
- Phone number search in OpenDental (fallback)

✅ **Patient Data Enrichment**: Analytics records now include full patient information from OpenDental

✅ **Automated Commlogs**: Call summaries are automatically posted to the patient's commlog in OpenDental



### 4. Check OpenDental
1. Open patient record
2. View Commlogs section
3. Find automated call summary entry

## OpenDental API Endpoints Used

1. **Patients GET (single)** - `GET /api/v1/patients/{PatNum}`
   - Fetches complete patient details by PatNum

2. **Patients GET (multiple)** - `GET /api/v1/patients?Phone={phone}`
   - Searches for patients by phone number

3. **Commlogs POST (create)** - `POST /api/v1/commlogs`
   - Creates automated call summary in patient's commlog

## Error Handling

The system is designed to **never fail** analytics processing:

- Patient not found? → Analytics stored without patient data ✅
- OpenDental API error? → Error logged, analytics still stored ✅
- Commlog creation fails? → Analytics stored, error logged ✅

## Performance

- Patient search: ~500-1000ms
- Patient fetch: ~300-500ms  
- Commlog creation: ~500-800ms
- **Total added time: ~1.5-2.5 seconds**
- Lambda timeout: 60 seconds (plenty of buffer)

## Troubleshooting

### Patient Not Found
1. Verify phone number format in OpenDental matches call data
2. Check CloudWatch logs for search attempts
3. Test OpenDental API directly (see testing guide)

### Commlog Not Created
1. Verify patient was found in logs
2. Check OpenDental API credentials
3. Test commlog API endpoint directly

See `docs/TESTING_PATIENT_INTEGRATION.md` for detailed troubleshooting guide.

## Architecture Overview

```
Call Completed
    ↓
Generate Analytics
    ↓
Search Patient (by PatNum or Phone)
    ↓
Fetch Patient Details from OpenDental
    ↓
Enrich Analytics with Patient Data
    ↓
Store Analytics in DynamoDB
    ↓
Create Commlog in OpenDental
```
